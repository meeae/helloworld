## 2. 배경 및 목적

### 2.1 배경
의사랑은 유비케어가 개발·운영하는 국내 대표 EMR 소프트웨어다. 현재 홈페이지(ysarang.com)는 오랜 기간 부분 업데이트만 이루어져 정보 구조가 노후화되었으며, 기존 고객(의사·간호사·실장)이 가장 자주 찾는 **제품 업데이트 정보**와 **주요 일정**이 고객센터 하위 메뉴에 숨겨져 있어 접근성이 매우 낮다.

### 2.2 핵심 문제
1. **제품 업데이트 게시판 탐색 불편** — 가장 중요한 정보가 고객센터 하위 depth에 위치
2. **주요 일정 접근성 부재** — 수가변경·청구마감 등 병의원 운영에 직결되는 일정이 메인에 미노출
3. **히어로 배너 마케팅 효과 미흡** — CTA 버튼 없음, 슬라이드 7개 이상으로 낭비
4. **제품소개서 노후화** — 기능 나열형 PDF 구성, 구매 의사결정 지원 요소 부재
5. **상단 유틸 메뉴 중복** — 원격문의/고객문의/제품가입문의/온라인 견적 4개 역할 불명확

### 2.3 목표
- 기존 고객의 **제품 업데이트 및 일정 정보 접근 시간 단축** (현재 3+ depth → 메인 직접 노출)
- 히어로 배너 **CTA 클릭률 향상** (현재 0 → 측정 가능한 구조로 전환)
- 신규 도입 문의 유입을 위한 **마케팅 랜딩 효과 개선**
- 전체 디자인 시스템 **일관성 확보** (컬러·타이포·컴포넌트 통일)

## 5. 기능 요구사항

### F-01. GNB 재편

**현재 구조**
```
[상단유틸] 원격문의 | 고객문의 | 제품가입문의 | 온라인 견적
[GNB] 회사소개 | 사업소개 | 제품소개 | 고객센터 | 미소센터 | 의약품검색
```

**개편 구조**
```
[상단유틸] 원격지원 | 온라인 견적                    [우측] 로그인 | 제품 도입 문의
[GNB] 회사소개 | 제품소개 | 제품 업데이트 | 주요 일정 | 고객센터 | 미소센터
```

**요구사항**
- `제품 업데이트` 메뉴를 GNB 1depth로 신설 (기존 고객센터 하위에서 격상)
- `주요 일정` 메뉴를 GNB 1depth로 신설
- 상단 유틸 메뉴를 `원격지원` / `온라인 견적` 2개로 단순화
- `제품 도입 문의` CTA 버튼을 GNB 우측에 Primary 버튼으로 배치
- 스크롤 시 GNB sticky 고정

---

### F-02. 긴급공지 띠배너 (Alert Bar)

**요구사항**
- 페이지 최상단에 1줄 고정 띠배너 배치
- 수가변경, 긴급 업데이트, 시스템 점검 등 시급 공지 노출
- CMS에서 on/off 및 내용 수정 가능
- 우측 X 버튼으로 세션 내 닫기 가능
- 공지 종류에 따라 색상 구분: 긴급(빨강) / 일반(주황) / 안내(파랑)

---

### F-03. D-day 주요 일정 스트립

**요구사항**
- GNB 하단에 가로 스크롤 가능한 일정 스트립 배치
- 표시 항목: 일정명 + D-day 숫자
- D-day 색상 규칙:
  - D-7 이하: 빨강 (`#C8290A`)
  - D-8 ~ D-14: 주황 (`#854F0B`)
  - D-15 이상: 초록 (`#3B6D11`)
- "전체 일정 보기 →" 링크로 주요 일정 페이지 이동
- 일정 데이터는 관리자 CMS에서 등록/수정

---

### F-04. 히어로 배너 개편

**현재 문제**
- 슬라이드 7개 이상, CTA 버튼 없음
- 배너 높이 과대로 하단 콘텐츠 미인지

**요구사항**
- 슬라이드 최대 4개로 제한
- 각 슬라이드에 필수 요소:
  - 배지 (NEW / 업데이트 / 이벤트)
  - 메인 헤드라인 (최대 25자)
  - 서브 카피 (최대 60자)
  - Primary CTA 버튼 (예: "업데이트 내용 보기")
  - Secondary CTA 버튼 (예: "사용 방법 가이드")
- 배너 우측에 해당 업데이트의 주요 변경사항 요약 카드 표시
- 배너 높이: 최대 320px (현재 약 420px에서 축소)
- 자동 슬라이드 간격: 5초, 호버 시 정지

---

### F-05. 메인 3단 콘텐츠 영역 재편

**현재 구조**: 공지&뉴스 / FAQ / 다운로드

**개편 구조**: 제품 업데이트 / 주요 일정 / 자주 묻는 질문

#### F-05-1. 제품 업데이트 패널
- 최신 4건 노출
- 각 항목 표시: 버전 태그(신규/업데이트/수정/긴급) + 제목 + 버전번호 + 날짜
- 태그 색상: 신규(빨강), 업데이트(파랑), 수정(초록), 긴급(주황)
- "전체 보기 +" 링크

#### F-05-2. 주요 일정 패널
- 향후 4건 노출 (지난 일정 자동 제외)
- 각 항목 표시: 날짜 + 일정명 + D-day 뱃지
- D-day 색상 F-03과 동일 기준 적용
- "전체 보기 +" 링크

#### F-05-3. 자주 묻는 질문 패널
- 최신/인기 4건 노출
- Q. 접두사 + 질문 제목
- "전체 보기 +" 링크

---

### F-06. 고객지원 바로가기 영역 개편

**현재 구조**: 4분할 (온라인 견적문의 / 원격지원 / 고객지원 / MiSO 지역센터) + 운영시간 박스

**개편 구조**: 5분할 균등 배치

| 항목 | 아이콘 | 설명 |
|------|--------|------|
| 원격 지원 | 💻 | 원격으로 빠른 해결 |
| 고객 지원 | 📞 | 전화·채팅 문의 |
| 제품 도입 문의 | 📄 | 신규 도입 상담 |
| MiSO 지역센터 | 📍 | 가까운 센터 찾기 |
| 의약품 검색 | 💊 | 약품 코드 조회 |

- 운영시간은 툴팁 또는 고객지원 항목 hover 시 표시 (별도 빨간 박스 제거)

---

### F-07. 제품소개 페이지 개편

**현재 문제**: PDF 다운로드 중심, 기능 나열형, 비교 요소 없음

**개편 방향**: 웹 기반 인터랙티브 랜딩 페이지

**페이지 구성 섹션**
1. **Hero** — 핵심 가치 제안 + 도입 문의 CTA
2. **주요 기능** — 기능별 카드 (아이콘 + 제목 + 설명 + 데모 GIF/스크린샷)
3. **도입 사례** — 병원 규모별 케이스 (의원/병원/종합병원)
4. **요금제 비교** — 플랜별 기능 비교표
5. **도입 프로세스** — 문의 → 상담 → 설치 → 교육 → 운영 단계
6. **CTA 마감** — 데모 신청 / 담당자 연결

**삭제**: PDF 다운로드 방식의 정적 제품소개서 (웹 페이지로 대체)

| Text Secondary | 그레이 | `#6B6B6B` |
| Border | 라이트 그레이 | `#E5E5E5` |

### 7.2 타이포그래피
| 역할 | 크기 | 굵기 |
|------|------|------|
| 히어로 헤드라인 | 28px | 500 |
| 섹션 타이틀 | 20px | 500 |
| 카드 타이틀 | 14px | 500 |
| 본문 | 14px | 400 |
| 캡션 / 날짜 | 12px | 400 |

### 7.3 컴포넌트 규칙
- 모든 카드: `border-radius: 10px`, `border: 1px solid #E5E5E5`
- 버튼 radius: `6px`
- Primary 버튼: `background: #D44A00`, `color: white`
- Ghost 버튼: `border: 1px solid #D44A00`, `color: #D44A00`
- 그림자: 사용 안 함 (flat design 유지)
- 아이콘: 이모지 또는 lucide-react 아이콘셋
