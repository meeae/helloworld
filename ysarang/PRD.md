## 2. 배경 및 목적

### 2.1 배경
의사랑은 유비케어가 개발·운영하는 국내 대표 EMR 소프트웨어다. 현재 홈페이지(ysarang.com)는 오랜 기간 부분 업데이트만 이루어져 정보 구조가 노후화되었으며, 기존 고객(의사·간호사·실장)이 가장 자주 찾는 **제품 업데이트 정보**와 **주요 일정**이 고객센터 하위 메뉴에 숨겨져 있어 접근성이 매우 낮다.

### 2.2 핵심 문제 (스크린샷 분석 기반)
현재 ysarang.com 메인 화면 분석 결과 확인된 문제:

1. **제품 업데이트 게시판 탐색 불편** — 가장 중요한 정보가 고객센터 하위 depth에 위치. 메인에서 3번 이상 클릭해야 접근 가능
2. **주요 일정 접근성 부재** — 수가변경·청구마감 등 병의원 운영에 직결되는 일정이 메인에 전혀 미노출
3. **히어로 배너 CTA 부재** — 슬라이드 7개 이상이나 CTA 버튼 없음. 마케팅 전환율 사실상 0%
4. **제품소개서 노후화** — 기능 나열형 PDF 구성, 비교·체험 요소 전무
5. **상단 유틸 메뉴 중복** — 원격문의/고객문의/제품가입문의/온라인 견적 4개 역할 불명확
6. **컬러 시스템 혼재** — 파랑(배너), 빨강(서비스센터 박스), 브랜드 오렌지 혼용으로 시각 일관성 붕괴

### 2.3 목표
- 기존 고객의 **제품 업데이트 및 일정 정보 접근 시간 단축** (현재 3+ depth → 메인 직접 노출)
- 히어로 배너 **CTA 클릭률 향상** (현재 0% → 측정 가능한 구조로 전환)
- 신규 도입 문의 유입을 위한 **마케팅 랜딩 효과 개선**
- 전체 디자인 시스템 **일관성 확보** (컬러·타이포·컴포넌트 토큰화)

## 4. 범위 (Scope)

### In Scope
- 메인 홈페이지 전면 개편 (레이아웃·컴포넌트·정보구조)
- GNB(글로벌 내비게이션 바) 재편
- 히어로 배너 영역 개편 및 CTA 추가
- 제품 업데이트 게시판 메인 노출 구조
- 주요 일정 D-day 위젯 신설
- 고객지원 바로가기 영역 개편
- 디자인 시스템 토큰화 (컬러·타이포·간격·컴포넌트)
- 반응형 레이아웃 (모바일/태블릿/데스크탑)

### Out of Scope
- 백오피스/CMS 시스템 개편
- 로그인 후 마이페이지 영역
- 모바일 앱
- 실제 게시판 데이터 마이그레이션

## 5. 기능 요구사항

### F-01. GNB 재편

**현재 구조**
```
[상단유틸] 원격문의 | 고객문의 | 제품가입문의 | 온라인 견적
[GNB] 회사소개 | 사업소개 | 제품소개 | 고객센터 | 미소센터 | 의약품검색
```

**개편 구조**
```
[상단유틸] 원격지원 | 온라인 견적                    [우측] 로그인 | 제품 도입 문의
[GNB] 회사소개 | 제품소개 | 제품 업데이트★ | 주요 일정★ | 고객센터 | 미소센터
```

**요구사항**
- `제품 업데이트` 메뉴를 GNB 1depth로 신설 (기존 고객센터 하위에서 격상)
- `주요 일정` 메뉴를 GNB 1depth로 신설
- 상단 유틸 메뉴를 `원격지원` / `온라인 견적` 2개로 단순화
- `제품 도입 문의` CTA 버튼을 GNB 우측에 Primary 버튼으로 배치
- 스크롤 시 GNB sticky 고정 (z-index: 50)
- 모바일에서 햄버거 메뉴로 전환

---

### F-02. 긴급공지 띠배너 (Alert Bar)

**요구사항**
- 페이지 최상단에 1줄 고정 띠배너 배치
- 수가변경, 긴급 업데이트, 시스템 점검 등 시급 공지 노출
- 우측 X 버튼으로 세션 내 닫기 (sessionStorage 활용)
- 공지 종류에 따라 색상 구분:
  - 긴급 → `bg-status-danger` (빨강)
  - 일반 → `bg-status-warning` (주황)
  - 안내 → `bg-status-info` (파랑)

---

### F-03. D-day 주요 일정 스트립

**요구사항**
- GNB 하단에 가로 스크롤 가능한 일정 스트립 배치
- 배경: `bg-brand-light` (`#FFF0E8`)
- 표시 항목: 일정명 + D-day 숫자
- D-day 색상 규칙:
  - D-7 이하 → `text-status-danger` (`#C8290A`)
  - D-8 ~ D-14 → `text-status-warning` (`#854F0B`)
  - D-15 이상 → `text-status-safe` (`#3B6D11`)
- 지난 일정(D-day 음수) 자동 필터링
- "전체 일정 보기 →" 링크

---

### F-04. 히어로 배너 개편

**현재 문제**
- 슬라이드 7개 이상, CTA 버튼 없음
- 배너 높이 과대(~420px)로 하단 콘텐츠 미인지

**요구사항**
- 슬라이드 최대 4개로 제한
- 배너 높이: 최대 320px
- 각 슬라이드 필수 요소:
  - 배지 (NEW / 업데이트 / 이벤트)
  - 메인 헤드라인 (최대 25자)
  - 서브 카피 (최대 60자)
  - Primary CTA 버튼 ("업데이트 내용 보기")
  - Secondary CTA 버튼 ("사용 방법 가이드")
  - 우측 변경사항 요약 카드 (최대 4항목)
- 자동 슬라이드 5초, 호버 시 정지
- 배경: 다크 네이비(`#1a1a2e`) — 브랜드 오렌지와 대비

---

### F-05. 메인 3단 콘텐츠 영역 재편

**개편 구조**: 제품 업데이트 / 주요 일정 / 자주 묻는 질문

#### F-05-1. 제품 업데이트 패널
- 최신 4건 노출
- 버전 태그: 신규(빨강) / 업데이트(파랑) / 수정(초록) / 긴급(주황)
- 표시: 태그 + 제목 + 버전번호 + 날짜

#### F-05-2. 주요 일정 패널
- 향후 4건 노출 (지난 일정 자동 제외)
- 날짜 + 일정명 + D-day 뱃지

#### F-05-3. 자주 묻는 질문 패널
- 최신/인기 4건 노출

---

### F-06. 고객지원 바로가기 영역

**개편 구조**: 5열 균등 배치

| 항목 | 설명 |
|------|------|
| 원격 지원 | 원격으로 빠른 해결 |
| 고객 지원 | 전화·채팅 문의 |
| 제품 도입 문의 | 신규 도입 상담 |
| MiSO 지역센터 | 가까운 센터 찾기 |
| 의약품 검색 | 약품 코드 조회 |

- 기존 빨간 박스 운영시간 제거 → 고객지원 hover 시 툴팁으로 표시

## 11. 제약사항 및 가정

- 기존 URL 구조 최대한 유지 (SEO 영향 최소화)
- 백엔드 CMS는 기존 시스템 활용
- D-day 데이터는 관리자 수동 등록으로 시작
- 브랜드 로고·컬러 아이덴티티 유지
- Pretendard 폰트 CDN 사용
